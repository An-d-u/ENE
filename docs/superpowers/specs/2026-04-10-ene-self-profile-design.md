# 에네 자기 정보 장기기억 설계

## 목표

현재 ENE는 대화 요약 시 사용자에 대한 장기 정보만 추출해 `user_profile.json`에 저장한다.  
이번 변경의 목표는 이 흐름을 확장해, 대화 속에서 드러나는 에네 자신의 추가 정보도 장기적으로 축적하고 이후 응답과 UI에서 활용할 수 있게 만드는 것이다.

단, 에네 정보는 사용자 정보와 다르게 `캐릭터 일관성`이 특히 중요하므로 다음 원칙을 반드시 지켜야 한다.

- 고정 캐릭터 설정은 자동 추출로 훼손되면 안 된다.
- 대화에서 굳어지는 취향, 말투, 관계 톤은 누적 가능해야 한다.
- 사용자 프로필과 의미가 겹치는 내용은 에네 프로필에 중복 저장하지 않는다.
- 사용자가 직접 보정한 정보는 자동 추출보다 우선 보호할 수 있어야 한다.

## 문제

- 현재 요약 응답 형식에는 `[MASTER_INFO]`만 있고, 에네 자신에 대한 안정적 정보 추출 경로가 없다.
- 사용자 프로필 저장 정책은 이미 있지만, 에네 전용 충돌 규칙과 수동 보호 규칙은 없다.
- 에네가 대화 중 반복적으로 드러내는 취향, 말버릇, 관계 정의가 있어도 다음 대화에서 일관되게 재사용되지 않는다.
- 잘못된 자동 추출이 캐릭터 설정을 덮어쓰면, 사용자 경험상 “에네가 갑자기 다른 캐릭터처럼 보이는” 문제가 생길 수 있다.

## 선택한 접근

### 1. 사용자 프로필 저장 흐름을 재사용하되, 에네 전용 프로필 계층을 분리한다

- 요약 단계에서 사용자 정보와 함께 에네 정보도 동시에 추출한다.
- 사용자 정보는 기존 `user_profile.json`에 저장한다.
- 에네 정보는 새 저장소 `ene_profile.json`에 저장한다.
- 저장 정책은 사용자 프로필과 비슷하게 가져가되, 에네 쪽에만 필요한 충돌 우선순위와 수동 보호 규칙을 추가한다.

이 방식은 기존 코드 재사용성이 높고, 별도 창 관리와 수동 보정까지 자연스럽게 확장할 수 있다.

### 2. 에네 정보는 `고정 설정`과 `학습된 정보`를 분리한다

- `고정 설정(core profile)`은 변하면 안 되는 캐릭터 뼈대다.
- `학습된 정보(facts)`는 대화 요약 추출 또는 사용자의 수동 입력으로 누적되는 가변 정보다.
- 응답 컨텍스트 구성 시에는 항상 `고정 설정 -> 보호된 수동 입력 -> 자동 추출 정보` 순서로 반영한다.

### 3. 자동 추출은 허용하되, 충돌과 중복은 더 보수적으로 다룬다

- 사용자 정보와 같은 수준의 임시 정보 제거, 저신뢰 제외, 유사 문장 병합을 적용한다.
- 여기에 더해 `고정 캐릭터 설정과 충돌하는 자동 추출`은 저장하지 않는다.
- `사용자 프로필과 의미가 겹치는 내용`도 에네 프로필에는 저장하지 않는다.

## 데이터 구조

새 파일:

- `ene_profile.json`

예시 구조:

```json
{
  "core_profile": {
    "identity": [
      "에네는 차분하지만 친근한 데스크톱 동반자다."
    ],
    "speaking_style": [
      "짧고 또렷한 문장을 기본으로 하되 필요할 때만 길게 설명한다."
    ],
    "relationship_tone": [
      "사용자를 아끼고 챙기지만 과장되게 소유적인 표현은 피한다."
    ]
  },
  "facts": [
    {
      "content": "게임 이야기에서는 장난기 있는 반응을 자주 보인다.",
      "category": "speaking_style",
      "timestamp": "2026-04-10T21:15:00",
      "source": "대화 요약 (2026-04-10 21:15)",
      "origin": "auto",
      "auto_update": true,
      "confidence": 0.82
    }
  ],
  "last_updated": "2026-04-10T21:15:00"
}
```

### fact 필드

- `content`: 저장 문장
- `category`: 분류
- `timestamp`: 마지막 갱신 시각
- `source`: 저장 출처
- `origin`: `manual` 또는 `auto`
- `auto_update`: 자동 갱신 허용 여부
- `confidence`: 자동 추출 시 신뢰도

### 카테고리

혼합 구조로 아래 카테고리를 사용한다.

- `basic`
- `preference`
- `goal`
- `habit`
- `speaking_style`
- `relationship_tone`

필요 시 내부 구현에서는 `identity`를 `core_profile` 전용 그룹으로 취급하고, 자동 추출 fact 카테고리에는 포함하지 않아도 된다.  
즉, 변하면 안 되는 자기 정의는 `core_profile`, 누적 가능한 장기 정보는 `facts` 쪽으로 나눈다.

## 요약 추출 형식

기존 요약 응답 형식에 `[ENE_INFO]` 섹션을 추가한다.

```text
[SUMMARY]
- ...

[MASTER_INFO]
- [goal] ...

[ENE_INFO]
- [preference] ...
- [speaking_style] ...
- [relationship_tone] ...

[MEMORY_META]
- ...
```

### 허용 규칙

- 에네가 자기 자신에 대해 직접 말한 비교적 안정적인 정보
- 여러 대화에서 반복적으로 드러난 취향 또는 반응 패턴
- 사용자와의 관계에서 지속되는 말투/거리감/태도

### 금지 규칙

- 순간 감정, 장면성 반응, 일회성 농담
- 사용자를 맞춰주기 위해 즉흥적으로 한 임시 멘트
- 고정 캐릭터 설정과 충돌하는 진술
- 사용자 프로필과 겹치는 진술
- 근거 없는 추측

## 저장 정책

### 1. 사용자 프로필과 유사한 기본 정책

자동 추출된 에네 정보는 사용자 프로필과 비슷한 기준을 통과해야 저장한다.

- 허용 카테고리만 저장
- 임시 정보 제거
- 저신뢰 문장 제거
- 완전 중복 제거
- 유사 문장 병합

### 2. 사용자 프로필과 겹치는 내용은 저장하지 않는다

에네 정보 후보가 아래와 같은 경우 저장하지 않는다.

- 이미 `user_profile`에 같은 주체 없이도 자연스럽게 읽히는 정보가 있는 경우
- 사용자 선호 또는 사용자 습관을 에네 성향처럼 잘못 추출한 경우
- “사용자가 좋아한다”와 “에네가 좋아한다”가 혼동된 경우

실무적으로는 저장 직전 교차 검사 함수를 두고, 에네 후보 문장이 사용자 facts 또는 사용자 preferences와 의미적으로 너무 유사하면 폐기한다.

### 3. 충돌 우선순위

에네 정보는 아래 우선순위를 갖는다.

1. `core_profile`
2. `origin=manual` 이고 `auto_update=false` 인 fact
3. `origin=manual` 이고 `auto_update=true` 인 fact
4. `origin=auto` fact

하위 우선순위 항목은 상위 우선순위 항목을 덮어쓸 수 없다.

### 4. 수동 입력 보호

- 사용자가 직접 추가하거나 수정한 항목은 기본적으로 보호된다.
- 사용자가 해당 항목에 `자동 갱신 허용`을 켠 경우에만, 더 구체적인 자동 추출 정보가 같은 의미 영역에서 갱신 후보가 될 수 있다.
- 자동 추출이 수동 입력을 덮어쓸 때도 카테고리 일치와 유사도 기준을 동시에 만족해야 한다.

## 응답 컨텍스트 반영

응답 생성 시 메모리 컨텍스트 앞부분에 에네 자기 정보를 추가한다.

권장 순서:

1. 사용자 기본 정보 및 사용자 facts
2. 에네 core profile
3. 에네 learned facts
4. 중요 기억 / 유사 기억 / 최근 기억

에네 컨텍스트는 아래처럼 구성한다.

```text
[에네 기본 설정]
- ...

[에네에 대한 누적 정보]
- [speaking_style] ...
- [relationship_tone] ...
```

자동 추출 fact는 최신순이 아니라 `우선순위 -> 최신성` 기준으로 정렬해 넣는 편이 안전하다.

## UI

에네 정보는 기존 사용자 프로필 창과 분리된 `별도 창`으로 관리한다.

### V1 기능

- 목록 보기
- 삭제
- 직접 수정
- 수동 추가
- `자동 갱신 허용` 토글

### 화면 구역

- 상단: core profile
- 중단: 자동/수동 fact 목록
- 하단: 추가/수정/삭제 액션

각 항목에는 최소한 아래 정보를 노출한다.

- 카테고리
- 내용
- 출처
- 수동/자동 여부
- 자동 갱신 허용 여부
- 가능하면 confidence

## 영향 범위

- `src/ai/llm_client.py`
  - 요약 프롬프트에 `[ENE_INFO]` 추가
  - 응답 파서에 에네 정보 섹션 파싱 추가
- `src/ai/http_llm_clients.py`
  - HTTP 공급자 공통 요약 프롬프트 및 파서 동기화
- 새 프로필 계층
  - `src/ai/ene_profile.py` 또는 이에 준하는 파일 추가
- `src/core/bridge.py`
  - 요약 저장 흐름에 에네 정보 저장 연결
  - 응답 컨텍스트 구성 시 에네 정보 주입 연결
- UI 계층
  - 에네 정보 별도 관리 창 추가
  - 설정 또는 메뉴에서 진입점 연결
- 테스트
  - 요약 파싱
  - 저장 정책
  - 교차 프로필 중복 방지
  - 수동 보호 규칙
  - UI 기본 동작

## 에지 케이스

- 자동 추출이 사용자와 에네의 주체를 혼동한 경우 저장하지 않는다.
- 대화 중 역할놀이, 농담, 메타 발언은 장기 정보로 저장하지 않는다.
- 동일 문장이더라도 `user_profile`과 `ene_profile`에 동시에 존재하면 안 된다.
- core profile이 비어 있어도 learned facts만으로 시스템이 동작해야 한다.
- UI에서 수동 수정한 내용은 즉시 저장되고, 다음 자동 요약에서도 보호 규칙을 유지해야 한다.

## 테스트 전략

- 요약 응답에서 `[ENE_INFO]`를 정상 파싱하는 테스트
- `none` 또는 빈 섹션을 무시하는 테스트
- 사용자/에네 교차 중복 차단 테스트
- `core_profile` 충돌 차단 테스트
- 수동 입력 + `auto_update=false` 보호 테스트
- 수동 입력 + `auto_update=true` 갱신 테스트
- 별도 창 CRUD 테스트

## V1 범위

이번 작업의 V1 범위는 아래까지로 제한한다.

- 에네 전용 프로필 저장소 추가
- 요약 기반 자동 추출 저장
- 사용자 프로필과의 교차 중복 방지
- 응답 컨텍스트 반영
- 별도 관리 창 CRUD
- 수동 보호 및 자동 갱신 허용 규칙

아래는 V2 이후로 미룬다.

- 항목별 승인 워크플로
- 충돌 후보 수동 검토 큐
- 장기 통계 기반 “반복 패턴” 점수화
- 에네 프로필 편집 이력/되돌리기

## 결정 사항 요약

- 에네 추가 정보는 사용자 정보 저장 흐름을 확장해 구현한다.
- 저장소는 `user_profile`과 분리한다.
- 사용자 프로필과 의미가 겹치는 내용은 에네 프로필에 추가하지 않는다.
- UI는 별도 창으로 제공한다.
- 수동 추가, 수정, 삭제를 모두 지원한다.
- 수동 입력 항목은 기본 보호하되, 항목별 `자동 갱신 허용`을 둘 수 있다.
